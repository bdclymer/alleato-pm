# Feature Status And Limitations

Use this skill when the user asks whether a feature exists, whether the
assistant can do something, or whether behavior is live, partial, planned,
blocked, or deprecated.

## Instructions

1. Use `search_feature_registry` or `get_feature_details`.
2. Treat published help articles as documented behavior.
3. Treat route-only entries as implementation evidence, not complete product
   documentation.
4. Never claim an assistant write action is executable unless metadata says so.
