## **Financial Markup Setup and Process**

The team clarified the updated financial markup process in Job Planner to streamline project setup and ongoing management.

- **Default Markup Settings Simplify Setup** (01:31)  
    - New projects now have default settings applying markups only to changes and linked budget codes automatically.  
    - Kristen explained users should not change these defaults to avoid errors.  
    - This update reduces manual setup work and errors for assistant PMs like La' Kisha starting new projects.  
    - The goal is to lower support requests by making markup handling more consistent and automated.

- **Key Markup Types and Filters Explained** (02:51)  
    - The main markups used are insurance (currently **1.35%**) and contractor fee (usually around **10%**).  
    - Filters should remain blank so markups apply broadly, avoiding errors from limited application to specific cost codes.  
    - The “applies to” section must be set manually to assign markups correctly in insurance and contractor fee budget fields.

- **Compounding Markup Option and Change Order Interaction** (05:23)

    - Once change requests link to contracts, financial markup editing locks and requires support intervention.  
    - Jesse confirmed the team primarily uses change requests, so this lock is relevant.
