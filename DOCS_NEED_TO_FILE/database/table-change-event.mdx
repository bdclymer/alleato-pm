# change_events table

| UI Field          | Type             | DB Table.Column                                                       | Notes                                                                                           |
| ----------------- | ---------------- | --------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| **Number**        | string / int-ish | `change_events.number`                                                | Usually unique per project (often displayed as `#002`). Store as string to preserve formatting. |
| **Title**         | string           | `change_events.title`                                                 | Required.                                                                                       |
| **Status**        | enum             | `change_events.status`                                                | e.g. `open`, `pending`, `closed` (whatever your app supports).                                  |
| **Origin**        | fk/enum          | `change_events.origin_id` **or** `change_events.origin`               | If you want Procore-style sources, make an `origins` lookup table.                              |
| **Type**          | fk/enum          | `change_events.type_id` **or** `change_events.type`                   | Procore examples: Allowance, Owner Change, etc.                                                 |
| **Change Reason** | fk/enum          | `change_events.change_reason_id` **or** `change_events.change_reason` | Procore examples: Allowance, Client Request, etc.                                               |
| **Scope**         | fk/enum          | `change_events.scope`                                                 | e.g. `in_scope`, `out_of_scope`, `tbd`.                                                         |
