  # List key domain entities from architecture, with table names and FK types
  - name: "{{EntityName}}"
    table: "{{table_name}}"

    fk_type: "{{INTEGER_or_UUID}}"

  # Extract key constraints and rules from architecture decisions
  - "{{constraint_from_architecture}}"

  # Extract design patterns chosen in architecture

**Instructions:** Populate each field by extracting from the architecture document sections — technology choices, data architecture decisions, design patterns, and constraints. Use actual values, not placeholders. If a field doesn't apply, omit it.

**Purpose:** PRP creation workflows consume this file as input instead of re-researching decisions that were already made during architecture. PRP still runs its own freshness checks (regenerating Supabase types, reviewing incident log).
