C-Suite Agent Architecture

A multi-agent system where a Chief Strategist routes questions to domain-specialist agents. Each agent has its own system prompt, tool set, and model config. All agents run server-side via Vercel AI SDK.

How to Wire a New Agent

// 1. Add config to agentRegistry in orchestrator.ts
agentRegistry.register({
  name: 'coo',
  model: 'openai/gpt-5.4-mini',
  systemPrompt: getCOOSystemPrompt,
  tools: createOperationalTools,
});

// 2. Add consultXxx tool in createStrategistTools()
consultCOO: tool({
  description: 'Consult the COO for operations, schedule, and vendor questions',
  execute: async ({ question }) => agentRegistry.invoke('coo', question),
});

// 3. Add name to AGENT_NAMES in agents/types.ts
export const AGENT_NAMES = ['strategist', 'cfo', 'coo', ...] as const;

// 4. Update routing rules in agents/strategist.ts

Known Limitations

Teams DM 403 Errors — Permanent, No Fix. 273 of 283 chats sync successfully. 10 permanent 403s on cross-tenant (@unq.gbl.spaces) and meeting chats (@thread.v2). Hard Microsoft Graph API limitation with app-only auth. Do not file a bug or attempt a fix.
No Review UI for Low-Confidence Assignments. Items below 0.70 confidence go to document_attribution_candidates with no frontend. Query directly: SELECT * FROM document_attribution_candidates ORDER BY created_at DESC LIMIT 50;

Embedding Config — tool-utils.ts (source of truth)

// Always use this — never call OpenAI SDK directly in tool code
const EMBEDDING = {
  LARGE: { model: "text-embedding-3-large", dimensions: 3072 },  // document_chunks, ai_memories, knowledge
  SMALL: { model: "text-embedding-3-small", dimensions: 1536 },  // conversation_memories (legacy)
};

// For new tools querying document_chunks
const embedding = await generateEmbedding(openai, query, EMBEDDING.LARGE);
const results = await supabase.rpc("search_document_chunks", {
  query_embedding: embedding,
  match_count: 10,
  category: "email",  // optional: 'email' | 'teams_message' | 'document' | 'meeting'
});

Adding a New RAG Tool — Checklist

1
Choose the right tool file
financial.ts, operational.ts, project-tools.ts, or a new file.
2
Read an existing tool for the pattern
tool({ description, parameters: z.object({...}), execute: async (...) => {...} })
3
Use createServiceClient() for DB access
Server-side only. Never call Supabase from a client component in tool code.
4
Use generateEmbedding() for vector search
generateEmbedding(openai, query, EMBEDDING.LARGE) then supabase.rpc("search_document_chunks", {...})
5
Add to the factory function's return object
e.g. createFinancialTools(), createOperationalTools()
6
Update the Strategist system prompt
Describe when to call the new tool in agents/strategist.ts routing rules.

Known Gaps

Confirmed missing features as of Phase 1 completion. Not hypothetical — verified against implementation.

COO, CHRO, CRO, VP BD not wired
Operational, people, risk, and business development questions fall through to Strategist with general-purpose tools. Prompts exist at frontend/src/lib/ai/agents/ — needs wiring only.
Phase 2+
No proactive insight generation
AI only answers questions. No automated budget/schedule/cash flow alerts. The ai_insights table schema is ready for Phase 2 insert operations.
Phase 2A
No notification system
Insights are stored in ai_insights but not routed to users via push, email, or Teams.
Phase 2B
No AI briefing card on dashboard
Users must open chat to see AI analysis. project_health_dashboard and project_issue_summary views exist and are ready to feed dashboard cards.
Phase 2C
Document auto-classification
Uploaded docs are ingested but not auto-classified by type (spec, submittal, drawing, etc.).
Phase 3A
Smart template pre-filling
RFI and change order forms have no AI pre-fill. Requires tool integration with form submission flow.
Phase 3D
Low-confidence email attribution UI
document_attribution_candidates review queue has no frontend. Admins must query the table directly.
Unscheduled

Debugging Bad Answers

1
Find the session in chat_history
Check metadata.tool_trace for which tools were called.
2
Wrong tool called?
Fix the Strategist prompt routing instructions first, not the SQL. File: agents/strategist.ts
3
Right tool, thin results?
Check table freshness: risks, ai_insights, document_metadata, project_health_dashboard.
4
Stale data?
Re-run ingestion for affected document metadata IDs.
!
Risk portfolio routing rule
Risk queries must call getProjectsWithRisks, NOT getPortfolioOverview. If wrong, fix agents/strategist.ts.
