Alleato RAG pipeline

From source artifact to assistant answer, project intelligence, and executive brief

This page explains the complete path for a meeting transcript, Teams message, email, synced document, or manual upload. The important split: live Q&A reads the vector store, while the product intelligence views read compiled cards and synthesized packets.

Ingest foundation Syncs create catalog rows, store raw text, assign projects, then chunk and embed.
Assistant path Every chat question retrieves live evidence from vectors, tools, SQL, and Acumatica.
Intelligence path Deep extraction creates evidence cards; synthesis creates current project and portfolio reads.

What each output is for

AI Assistant

Best for flexible questions. It retrieves relevant chunks and calls live tools at answer time, so it can combine source evidence with current project, finance, schedule, and task data.

Project Intelligence

Best for a stable operating read. It uses compiled evidence cards plus a rolling synthesized packet, so the page is not just search results.

Progress Log

Best for source traceability. It shows granular insight_cards from meetings, emails, and Teams messages with evidence quotes and links.

Portfolio Brief

Best for Brandon-level prioritization. It synthesizes across current project packets plus financial pulse data into what changed, what needs attention, and who is waiting.

Source-specific intake

Meeting transcript

Fireflies sync normalizes transcript markdown, uploads it to Storage, writes catalog/raw-text records, segments the meeting, embeds transcript chunks, and extracts tasks/risks/decisions.

Email

Outlook sync writes raw intake first, filters operational relevance, assigns a project when confidence is high enough, then embeds the raw content from the AI Database.

Teams

Graph sync captures channels and DMs, groups DMs into conversation documents, writes them into the catalog, and vectorizes them for assistant and intelligence retrieval.

Documents

OneDrive/SharePoint sync and manual uploads create document catalog rows, extract text where possible, hydrate raw text from the AI Database, then chunk and embed.
