Goal

Feature Goal: Enable bidirectional synchronization between Alleato PM's scheduling tool and Microsoft Project for the Web (Dataverse-based), allowing users to connect their Microsoft account, import project schedules, and push task updates back.

Deliverable: OAuth2 connection flow, Dataverse API client, sync service, and UI components for connecting, importing, and syncing MS Project data with Alleato's schedule_tasks table.

Success Definition: User can authenticate via Azure AD, browse MS Project projects, pull tasks + dependencies + milestones into Alleato, and push updates back.

Why

  • Construction firms commonly use Microsoft Project for scheduling
  • Manual CSV export/import is error-prone and creates stale data
  • Teams split between MS Project (schedulers) and Alleato (project managers) need a single source of truth
  • Extends existing Schedule tool with live external data source
  • Follows established Acumatica integration pattern

Known Pitfalls & Prevention

OperationSet is the ONLY write path
Direct PATCH/POST to msdyn_projecttask returns 403/500. ALL writes must go through CreateOperationSetV1 → PssCreate/PssUpdate → ExecuteOperationSetV1. Limit: 200 ops per set.
FK Type Mismatch (3+ historical bugs)
projects.id is INTEGER. Any FK column referencing it must be INTEGER, never UUID. Verify after migration with npm run db:types.
Token Scope Must Match Org URL
A token for globaldisco.crm.dynamics.com won't work for contoso.crm.dynamics.com. After discovery, acquire a new token scoped to the specific org.
msdyn_effort is hours, not days
Divide by 8 (hours per workday) when converting to duration_days. Treating hours as days will cause wildly wrong durations.
Outline Level ≠ Parent Task
Use _msdyn_parenttask_value for hierarchy, not msdyn_outlinelevel. Outline level only indicates depth.
