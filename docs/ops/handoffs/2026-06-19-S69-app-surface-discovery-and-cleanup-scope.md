## Known Pitfalls

- **Do NOT trust the raw orphan-audit counts.** They over-flag barrel exports (`useProjects`, `useUsers`, `useCompanies` were flagged — deleting them breaks the app) and the `ds/` palette.
- **Verify before deleting** (CLAUDE.md rule): grep for dynamic/string references, barrel re-exports, and feature-in-progress before removing any "orphan." Empty + has consumers = incomplete feature, not dead feature.
- The repo has **concurrent sessions + an auto-committer** that mutate the tree live; re-run typecheck for ground truth and expect your staged files to occasionally be swept into another session's commit.
- `npm run typecheck` has a 60s bound that times out on the full program; use `TYPECHECK_NO_TIMEOUT=1 npx tsc --noEmit` for a definitive result. Delegate it to a sub-agent (never run in the main thread).
- The headless chat **input** cannot be driven (React 19 controlled state); verify assistant behavior via `POST /api/ai-assistant/chat` from the page context instead. Session `id` and message `id` must be UUIDs (`crypto.randomUUID()`).
- Editing `page-descriptions.json` or adding/removing a route/tool requires `npm run map:project` or the project-map pre-commit gate fails.
