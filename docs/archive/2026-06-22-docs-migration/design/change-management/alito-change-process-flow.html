Status States

  • Change Event: Draft → Open → Grouped
  • PCO: Draft → Submitted → Under Review → Approved / Revision
  • Official CO: Draft → Pending Signatures → Fully Executed

Role Permissions

  • Super Can create Change Events only
  • PM Can create PCOs, send to client, manage revisions
  • PM action auto-notifies VP + leadership group
  • Client Sees simplified view, approves or requests revision
  • Leadership sees headline view on dashboard and at login

What's Never Lost

  • Change Events are permanent — never deleted
  • PCO revisions are versioned, never voided
  • All linked documents persist across lifecycle
  • Full comment + notification history preserved

Key Design Rules

  • No partial approvals — all or nothing
  • Multiple change events can group into one PCO
  • Sub COs can run parallel to prime approval in real-world mode
  • Role-based views control information density
  • Every step links to source documents

Notification Triggers

  • PCO created → email to VP + distribution
  • PCO created → project dashboard panel
  • PCO created → login notification queue
  • Client approves/rejects → PM notified
  • RFQ pricing returned → PM notified
  • CO fully executed → leadership + PM
