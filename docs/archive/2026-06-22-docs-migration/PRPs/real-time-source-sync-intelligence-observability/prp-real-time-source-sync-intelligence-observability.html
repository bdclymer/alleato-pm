Goal

Build a reliable, near-real-time ingestion and observability system for meeting transcripts, Outlook emails, email attachments, Teams messages, OneDrive/SharePoint documents, extracted tasks, vectorized chunks, source intelligence jobs, and project intelligence packets.

The intended product outcome is fresh AI Assistant project intelligence with clear sync, vectorization, compiler, task, packet, and error visibility.

Recommended Architecture

Source change from Microsoft Graph / Fireflies
  -> public webhook or bounded poll trigger
  -> source event ledger / sync state update
  -> backend queue job
  -> source-specific delta fetch or transcript fetch
  -> document_metadata / project_documents / attachment rows
  -> parse / text extraction / project attribution
  -> document_chunks + embedding status
  -> task extraction + source_signal_candidates
  -> insight_cards + packet_refresh_jobs
  -> intelligence_packets
  -> AI Assistant packet-first answers + exact source lookup
  -> admin health dashboard + alerts

Use webhooks and delta sync to wake the backend. Use cron for renewal and reconciliation. Do not run heavy sync or vector work inside Vercel request paths.

Required Outcome

  • One admin page shows last sync, last success, last error, backlog, vectorization, compiler, task extraction, and packet freshness.
  • Every source has durable state and alertable failures.
  • AI Assistant can explain stale or missing evidence by stage instead of silently returning thin answers.

Database Schema

Supabase types were regenerated with:

npx supabase gen types typescript --project-id "lgveqfnpkxvzbnnwuled" --schema public > frontend/src/types/database.types.ts

Critical FK rules: projects.id is number/integer; people.id is UUID/string; document_metadata.id is UUID/string; document_chunks.chunk_id is the chunk identifier.

Primary existing tables: graph_sync_state, document_metadata, document_chunks, fireflies_ingestion_jobs, source_intelligence_jobs, source_signal_candidates, packet_refresh_jobs, intelligence_packets, insight_cards, project_documents, and tasks.

Recommended additive schema: source_sync_runs, source_sync_health_snapshots, and graph_subscriptions.

Known Pitfalls

  • Never create UUID project_id columns.
  • Never add generic [id] app routes.
  • Never use Vercel cron as the worker for heavy processing.
  • Never claim verification without command output or browser artifacts.
  • Never hide configured provider/source failures behind fallback success.
