Guardrails for Resync from Updated Estimate

  • Allowed only when prime contract is draft or out for signature / pending.
  • Disabled once executed so executed SOV values cannot be mutated casually.
  • Show the estimate version the contract was built from.
  • Log every sync event: previous value, new value, user, timestamp, estimate revision.
  • If budget lines already exist, ask user to merge, replace, or cancel.

Recommended Implementation Sequence

  1. Create Prime Contract from Estimate UX + API.
  2. Seed Budget from Estimate.
  3. Add resync/version guardrails + change history.
  4. Add prime_contract_id to commitments.
  5. Add proper budget_line_id / cost_code_id FKs.
  6. Clean up V1 estimate model.
  7. Clean up legacy SOV tables last.
