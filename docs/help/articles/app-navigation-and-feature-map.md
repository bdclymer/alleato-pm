# Navigate Alleato PM

Alleato PM is organized around two kinds of pages: global pages and
project-scoped pages.

Global pages are not tied to one project. Use them for portfolio work,
company-wide directory records, account settings, knowledge, and the AI
assistant.

Project-scoped pages include a project ID in the route. Use them for work that
belongs to a single project, such as budget, commitments, change events, RFIs,
submittals, drawings, daily logs, documents, photos, schedule, and project
directory.

## Ask Alleato For The Right Page

Ask Alleato can answer navigation questions like:

- Where do I create a change event?
- Which page has project contacts?
- Where can I see project documents?
- Is this an admin page or a normal project page?

When possible, the assistant should include the exact route and a short reason
why that page is the right place.

## Current Boundary

The app sitemap and feature registry are generated from the codebase and help
article metadata. If a route exists but does not have a curated help article
yet, Ask Alleato should say that it found the route from the app inventory and
that user-facing documentation still needs to be filled in.
