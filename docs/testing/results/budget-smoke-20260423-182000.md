## Secondary Finding — Missing UUID Validation (Bug)

Routes with path parameters `[lineId]` and `[viewId]` do not validate UUID format before querying Postgres.

**Affected routes:**
- `GET /api/projects/67/budget/lines/[lineId]`
- `GET /api/projects/67/budget/lines/[lineId]/history`
- `GET /api/projects/67/budget/views/[viewId]`

**Behavior:** Passing a non-UUID string (e.g. `1`) causes Postgres to throw `invalid input syntax for type uuid` → the route returns 500 instead of 400/404.

**Expected:** Routes should validate UUID format upfront and return 400 (bad request) or 404 (not found).

**Fix:** Add UUID regex check at the top of each route handler before any DB query. Already flagged for fix.
