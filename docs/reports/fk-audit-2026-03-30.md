## Results by Form

### ChangeEventForm — CRITICAL ISSUES

**File:** `frontend/src/components/domain/change-events/ChangeEventForm.tsx`
**Submit page:** `frontend/src/app/(main)/[projectId]/change-events/new/page.tsx`
**API:** `frontend/src/app/api/projects/[projectId]/change-events/[changeEventId]/line-items/route.ts`

| Field | DB Column | FK Target | Dropdown Source | Status |
|-------|-----------|-----------|----------------|--------|
| Budget Code | `change_event_line_items.budget_code_id` | `budget_lines.id` | `project_cost_codes.id` | **MISMATCH — NOT SENT** |
| Vendor | `change_event_line_items.vendor_id` | `companies.id` | `vendors.id` | **MISMATCH — NOT SENT** |
| Contract | `change_event_line_items.contract_id` | `prime_contracts.id` | prefixed string `po-UUID` / `sub-UUID` | **MISMATCH — NOT SENT** |
| Commitment Line Item | `change_event_line_items.commitment_line_item_id` | (no FK) | `change_event_line_items.id` | **NOT SENT** |

**Root cause:** `new/page.tsx` lines 110–135 POST line items to the API but strip `budgetCodeId`, `vendorId`, `contractId`, and `commitmentLineItemId` from the payload.

The form *collects* these values correctly and the API *has* FK resolution logic to handle the `project_cost_codes` → `budget_lines` mapping and `vendors` → `companies` mapping — but the page handler never passes them, so all four fields arrive as `undefined`.

**Impact:** Budget code, vendor, contract, and commitment line item are silently discarded on every change event save. They appear empty in the detail view even though the user selected them.

**Fix required:**

In `frontend/src/app/(main)/[projectId]/change-events/new/page.tsx`, update the line item POST body (lines ~110–135):

```diff
const lineItemPayload = {
  description: lineItem.description,
  quantity: lineItem.quantity,
  unitCost: lineItem.unitCost,
  unitOfMeasure: lineItem.unitOfMeasure,
  costRom: lineItem.costRom,
  revenueRom: lineItem.revenueRom,
  nonCommittedCost: lineItem.nonCommittedCost,
  sortOrder: lineItem.sortOrder,
+ budgetCodeId: lineItem.budgetCode ?? undefined,
+ vendorId: lineItem.vendor ?? undefined,
+ contractId: lineItem.contract
+   ? lineItem.contract.replace(/^(po-|sub-)/, "")
+   : undefined,
+ commitmentLineItemId: lineItem.commitmentLineItemId ?? undefined,
};
```

Also check the edit flow in `frontend/src/app/(main)/[projectId]/change-events/[changeEventId]/page.tsx` for the same missing fields in PATCH requests.
