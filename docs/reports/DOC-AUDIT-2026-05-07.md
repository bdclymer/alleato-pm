## Section 5: Missing docs

The following reference documentation **should exist** for a Next.js 15 + Supabase + Python FastAPI + AI/RAG platform of this size, but doesn't:

### Critical gaps

1. **Backend (FastAPI) developer reference** — `docs/api/` covers only Next.js API routes. There is **no documentation** of:
   - `backend/src/` directory structure and module purposes
   - FastAPI endpoint reference (the equivalent of `MASTER-ROUTE-INDEX.md` for Python)
   - Render deployment runbook (the 3 cron jobs in CLAUDE.md aren't documented anywhere in `docs/`)
   - `backend/src/services/` service registry
   - How to add a new background job / cron service

2. **Single canonical onboarding doc** — `onboarding/` has `WELCOME_ONBOARDING_SPEC.md` (a feature spec for an in-app widget, not a developer guide). `project-overview/development-guide.md` is the closest, but no doc says "if you just cloned this repo, do these 10 things." A new engineer would not know to read CLAUDE.md, then design/, then patterns/.

3. **Environment variable reference** — Scattered across `SECURE-API-KEY-SETUP.md` and CLAUDE.md mentions. No single `docs/env-vars.md` listing every `.env` key, what owns it, where it's used, and what happens if it's missing. (Gate 17 in CLAUDE.md exists *because* this gap caused build crashes.)

4. **Render deployment + cron infrastructure** — CLAUDE.md memory says "set up 2026-05-06 — DO NOT move to GitHub Actions" with 3 cron jobs. None of this is in `docs/`. Should be `docs/deployment/render.md`.

5. **Vercel deployment runbook** — Same gap. Memory mentions staging/production split but there's no `docs/deployment/vercel.md` describing branch-to-environment mapping, env var management per environment, build gate checks, rollback procedure.

6. **Supabase operations runbook** — No doc on: backups, migration rollback procedure, RLS policy review process, service-role-key rotation, vector index maintenance, when/how to regenerate `database.types.ts`.

7. **AI/RAG architecture single source of truth** — Currently spread across `ai-plan/AI-MASTER-PLAN.md`, `ai-plan/AI-CSUITE-ARCHITECTURE.md`, `ai-plan/rag-pipeline/rag-pipeline.md`, `features/AI_KNOWLEDGE_BASE.md`, `features/MEMORY-SYSTEM.md`, `features/teams-compiler.md`, `project-overview/rag-and-ai-tools.md`, `architecture/COMMUNICATIONS-DATA-PIPELINE.md` (the latter referenced from CLAUDE.md memory but **doesn't exist in docs/**). Needs consolidation into one `docs/architecture/ai-rag.md` with the others archived.

8. **Communications data pipeline doc** — CLAUDE.md memory references `docs/architecture/COMMUNICATIONS-DATA-PIPELINE.md` as "AUTHORITATIVE" but the file doesn't exist. This is a documentation-vs-reality mismatch.

9. **MCP server / agent skills inventory** — The repo has dozens of registered skills (visible in tool list), no `docs/agents/skills.md` cataloging them.

10. **Glossary** — Domain terms (PCO, CO, RFI, submittal, retainage, SSOV, prime contract, change event vs change order) appear everywhere with no single definition reference.

### Nice-to-have gaps

- **Architecture decision log** — `ops/adr/` has only the template + README. Zero actual ADRs written.
- **Component inventory with screenshots** — `project-overview/component-inventory.md` is text-only. The design system has a Storybook-equivalent gap.
- **Database ER diagrams** — `design/entity-relationships/` exists but is sparse. Mermaid diagrams of the core schema would help.
- **Performance budget / SLO doc** — No defined response-time targets, no perf testing guide.
- **Security model doc** — RLS is mentioned everywhere; no single doc explains the auth + RLS model end-to-end.
- **Incident response runbook** — `RUNBOOK.md` exists at root but covers dev workflow, not production incidents.
