01
Surface Elevation
Use background depth — not borders — to separate regions
The rule: Adjacent sections at the same hierarchy level should NOT have borders between them. Use progressive background lightness instead. Think of it as Z-axis depth, not X/Y separation.
Layer 0
#09090b
Page background. Nothing sits below this.
Layer 1
#111113
Sidebar, secondary panels, subtle section backgrounds
Layer 2
#18181b
Cards, modal bodies, inline editors
Layer 3
#27272a
Hover states, selected rows, nested panels
Layer 4
#3f3f46
Active controls, input backgrounds on light surfaces
✗ Wrong
Section A
Heavy border around everything
Section B
Same style — identical to Section A
Section C
Every section looks the same
All sections share identical visual treatment. Nothing communicates importance. The UI feels like Tailwind cards stacked on top of each other.
✓ Right
Primary Layer
Sidebar / secondary content at Layer 1
Raised Panel
Content floats at Layer 2
Nested item at Layer 3
Elevation communicates hierarchy without a single unnecessary border. The eye reads the depth naturally, just like real Z-axis layering.

Bento Grid Layout
Variable cell sizes communicate relative importance. Not every cell is equal.
The key insight: A 1px gap with a single background color creates the grid lines. Cells share the same surface color but their SIZE communicates priority. This is how Vercel, Linear, and Supabase build dashboards.

05
Dividers — Used Sparingly
Rules are for within-section separation, never between major sections
✗ Overused borders
Dashboard
Section A content
Section B content
Section C content
Lines everywhere creates visual noise and makes everything feel equally important — defeating the purpose of hierarchy.
✓ Correct usage
Profile Settings
Display name, avatar, and personal info
Notifications
Email alerts and in-app preferences
Security
Password, sessions, and 2FA
Single-pixel dividers (7% opacity) separate logical groups within a section. Space above each label creates the visual break. Clean, readable, intentional.
06
Accent & Indicator Bars
Color signals meaning — category, status, urgency — without decorating everything
Active
Left border accent
border-left: 2px solid accent. Used for active state, selected item, or category marker. Subtle but immediately scannable.
Healthy
Top status stripe
border-top: 2px solid green. Used for status indicators on panels — operational, success, completed states.
Info
Tinted background callout
8% opacity accent background with matching border. Used for important notices, onboarding hints, feature announcements.
Warning
Status-colored callout
Same pattern with amber. Never use status colors decoratively — only when the state is genuinely warning, degraded, or requires action.
Rule: Status colors (green, amber, red) appear on less than 5% of the total UI surface area. When everything is colored, nothing is important.

15
Spacing System
Every value is a multiple of 4. Prefer multiples of 8.
4px
Icon-to-label, badge padding, micro gaps
8px
Tight inline spacing, icon margins
12px
Compact form fields, small component internals
16px
Standard padding — buttons, inputs, nav items
24px
Card/panel internal padding — default
32px
Between related groups within a section
48px
Between major sections on a page
64px
Page top/bottom padding, hero sections
Never use: 5px, 7px, 9px, 14px, 18px, 22px, 28px, 36px — these are not on the grid and create visual inconsistency that designers immediately notice.
16
Color System
Opacity-based text tiers. One accent. Status colors only for actual status.
Text Colors
Primary
rgba(255,255,255,0.95)
Headings, values, active labels
Secondary
rgba(255,255,255,0.65)
Body text, descriptions, regular UI copy
Tertiary
rgba(255,255,255,0.40)
Section labels, captions, eyebrows
Disabled
rgba(255,255,255,0.25)
Placeholder text, truly inactive
Status Colors (use sparingly)
Success
#22c55e
Active, live, complete, positive delta
Warning
#f59e0b
Pending, degraded, action required
Error
#ef4444
Failed, error, destructive, negative delta
Accent
#7c3aed
ONE primary: CTAs, focus rings, active nav
Anti-pattern: Never use status colors for decoration. If it's not actually green/amber/red in meaning, it should be the accent color or a neutral — not a fake success state.
