Visual Reference — Light Mode

Premium Light UI
Examples for AI Agents

The same 16 hierarchy techniques and layout patterns — recalibrated for light mode. Different rules, different traps, different wins.

Light mode is not dark mode inverted. Four key differences: (1) Elevation goes lighter — white panels float above a gray canvas, not the other way around. (2) Shadows carry the depth work that dark contrast does in dark mode — use them deliberately. (3) Text opacity values shift: secondary is 55% (not 65%), tertiary is 35% (not 40%). (4) Status and accent colors need to be slightly more saturated to hold against white backgrounds.
01
Surface Elevation — Light Mode
White is elevated. The page canvas is slightly gray. Panels float above it.
Critical inversion: In dark mode, deeper = darker. In light mode, elevated = whiter. Your sidebar and cards should be #ffffff — the background behind them is the gray. This is how Notion, Linear light, and Vercel light all work.
Layer 0 — Canvas
#f0f0f3
Page background. Cool gray — never pure white. Everything sits on top of this.
Layer 1 — Panels
#ffffff
Sidebar, cards, modals. Pure white = visually elevated off the canvas.
Layer 2 — Nested
#f8f8fb
Backgrounds within panels. Slightly gray = recessed from white.
Layer 3 — Active
#ededf0
Hover states, selected rows, focused regions.
Layer 4 — Controls
#e2e2e6
Input backgrounds, pressed states, active toggles.
✗ Wrong — white page, gray cards

02
Shadow System ✦ Light Mode Exclusive
Shadows do the depth work that contrast does in dark mode. This section doesn't exist in dark mode.
The rule: In light mode, shadows are not decorative — they are structural. A white card on a gray background with no shadow reads as flat. The same card with shadow-sm reads as elevated. Always use at minimum shadow-sm on white surface components.
None
no shadow
Only for elements already inside a raised container. Never for standalone panels.
Shadow SM
0 1px 2px rgba(0,0,0,.06)
0 1px 3px rgba(0,0,0,.04)
Default for all white surface panels, cards, inputs, nav items.
Shadow MD
0 4px 6px rgba(0,0,0,.06)
0 2px 4px rgba(0,0,0,.04)
Dropdowns, popovers, bento cells, KPI blocks, active cards.
Shadow LG
0 10px 15px rgba(0,0,0,.07)
0 4px 6px rgba(0,0,0,.04)
Modals, command palettes, large overlays. Reduce border opacity when using this.

07
Accent & Indicator Bars — Light Mode
Same pattern — but status colors need more saturation to hold against white
Active
Left border accent
3px solid accent. The card behind it is white with shadow-sm. Border provides category or active state signal without overwhelming the content.
Healthy
Top status stripe
2px solid green on top. Green needs to be #16a34a (not #22c55e) in light mode — the darker shade holds better contrast against white backgrounds.
Info
Tinted background callout
7% opacity accent background with 20% border. These subtler values work because the text color itself carries the accent — the bg just reinforces it.
Warning
Warning callout
Amber at #d97706 (not #f59e0b). Darker tone maintains contrast ratio on white. Always include an explicit label — color alone is never enough.

17
Color System — Light Mode
All opacity values reference rgba(0,0,0,x). Status colors are darker/more saturated than dark mode equivalents.
Text Colors — dark on white
Primary
rgba(0,0,0,0.88)
Headings, values, active labels
Secondary
rgba(0,0,0,0.55)
Body text, descriptions. 55%, not 65%.
Tertiary
rgba(0,0,0,0.38)
Eyebrows, captions, section labels
Disabled
rgba(0,0,0,0.22)
Placeholders, truly inactive elements
Status Colors — more saturated for white contrast
Success
#16a34a (not #22c55e)
Darker green holds against white backgrounds
Warning
#d97706 (not #f59e0b)
Darker amber — lighter would wash out
Error
#dc2626 (not #ef4444)
Darker red — better contrast ratio on white
Accent
#5b21b6 (not #7c3aed)
Deeper purple — maintains AA contrast on white
The saturation rule: Light mode status colors should be 1–2 shades darker than their dark mode equivalents. What reads clearly as #22c55e on a dark background needs to be #16a34a to maintain the same visual weight on white. Always verify WCAG AA contrast ratios (4.5:1 for normal text).
