## Form: submittal_create_edit

Title: Create Submittal / Edit Submittal
Component: `SubmittalFormPage` (`frontend/src/features/submittals/submittal-form-page.tsx`)

### URL Path
- Create: `/67/submittals/new`
- Create from package: `/67/submittals/new?package_id=<uuid>` (prefills package)
- Create from spec: `/67/submittals/new?specification_section=<section>` (prefills spec)
- Edit (page): `/67/submittals/<submittalId>/edit`
- Edit (inline): on detail page `/67/submittals/<submittalId>` → ⋯ menu → Edit (renders same form inline, `mode="inline"`)

### How to Open
- Create: list header → "Create Submittal" dropdown → "Create new submittal" (or "Create from package" / "Create from specification" which open picker dialogs first).
- Edit: detail page → ⋯ (MoreHorizontal) menu → Edit. Or navigate to `/edit` route directly.

### Fields
| Field | Control | Source | Stores to column | FK target |
|-------|---------|--------|------------------|-----------|
| submittal_number * | Input text | — | submittals.submittal_number | — |
| revision * | Input number | — | submittals.revision | — |
| title * | Input text | — | submittals.title | — |
| specification_section | Input text | — | submittals.specification_section | — |
| division | Input text | — | submittals.division | — |
| submittal_type_id | Select | `/api/projects/{id}/submittal-types` | submittals.submittal_type_id | submittal_types |
| status | Select (Draft/Open/Distributed/Closed) | enum | submittals.status | — |
| submittal_package_id | Select | `/api/projects/{id}/submittals/packages` | submittals.submittal_package_id | submittal_packages |
| responsible_contractor_id | Combobox (RHFComboboxField) | `useProjectCompanies` (project_companies junction; `value=c.id`, label=`c.company.name`) | submittals.responsible_contractor_id | **companies.id (numeric, no DB FK)** 🚩 |
| received_from_id | Combobox | `useAuthUsers` (auth user id) | submittals.received_from_id | auth user / people |

| final_due_date | Input date | — | submittals.final_due_date | — |

| is_private | Checkbox | — | submittals.is_private | — |

Auto-behaviors: on new, submittal_manager defaults to current user; ball_in_court defaults to manager when status=Draft; final_due_date = required_on_site_date − lead_time; lead_time auto-set from spec section division.

### Submit Action
- Create: `useCreateSubmittal` → `POST /api/projects/67/submittals`. On success redirect to new detail page.
- Edit: `useUpdateSubmittal` → `PUT /api/projects/67/submittals/<id>`. On success redirect to detail (or inline onSaved → refresh).
- Toast on success/error via hook (`appToast`).

### Success Criteria
- [ ] Create: new submittal appears in All Items tab / its package group.
- [ ] Redirect lands on detail page showing entered title/number.
- [ ] Edit: navigate away, re-open Edit → **all dropdowns pre-fill** (submittal type, package, status, responsible contractor, received from, submittal manager, ball in court).
- [ ] Responsible Contractor persists and shows correct company name on detail (Details tab).
- [ ] Submittal Manager / Received From persist and resolve to names on detail.
- [ ] Status change persists and renders in StatusBadge.
- [ ] Success toast appears.

### Cleanup
- Delete created submittal (⋯ → Delete → soft-delete to Recycle Bin). Then permanently leave in recycle bin or restore+delete. No hard-delete endpoint exists.

### Risk Notes
- 🚩 **FK/type mismatch (responsible_contractor_id).** Form schema types `responsible_contractor_id` as `z.string()`, and the combobox value is the **project_companies junction row id** (`c.id`), label from `c.company.name`. But the POST route schema is `z.coerce.number().int()` and the GET resolves the id against the **`companies` table** (`companyMap` keyed by `companies.id`). If `c.id` (junction id) ≠ `companies.id`, the saved value will not resolve to a company name on read, and z.coerce.number on a UUID string would NaN/fail. **Verify: create with a Responsible Contractor, reopen Edit and view Details — does the company name show?** This is the classic dropdown-source ≠ FK-target bug class. Confirm whether `useProjectCompanies` returns `c.id = companies.id` or junction id before concluding.
- received_from_id/submittal_manager_id are `z.string().uuid()` in the route but the form may send empty string → mapped to null in onSubmit (OK).
- No explicit error surface beyond toast; create/update mutations do show error toast with message (not swallowed).

6. **table_row_delete / bulk delete** — confirm UnifiedTablePage wraps row/bulk delete in a confirm.
